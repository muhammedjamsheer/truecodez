ViewChild and ViewChildren

The @ViewChild and @ViewChildren decorators used to access and manipulate DOM elements, directives and components.

We can use ViewChild to query one element from the DOM and ViewChildren for multiple elements as a QueryList of items.

Syntax

To Query a DOM element(s), we must supply the query selector, which can be a string or a type as the first argument to the ViewChild or ViewChildren

The argument static determines whether the query is performed, before or after the change detection.

static option is set to true when you want to access the ViewChild in ngOnInit. static option : false can only be accessed in ngAfterViewInit.or when the child is rendered dynamically . For Example inside a ngIf or ngSwitch etc.

The read option allows us to query a different token rather than the default and is useful when the element is associated with multiple types.
