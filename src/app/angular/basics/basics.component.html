Project Folder Structure

1.e2e - This folder is for an end to end testing purposes. It contains the configuration files related to performing the unit test of the projects.

2.node_modules - This folder contains the downloaded packages as per the configuration.

3.src - Tt is the folder which contains the main code files related to our angular application. It contains 3 subfolders as

  • app - App folder contains the Angular project-related files like components, HTML files, etc.
  • assets - Assets folder contains any static files like images, stylesheets, custom javascript library files etc.
  • environments - Environments folder contains the environment-related files which are required during deployment or build of the projects..
3 different configuration files

1.tsconfig.json - If tsconfig.json files exist within the project root folder, that means that the project is a basically TypeScript project

2.package.json - package.json is basically a JSON file that contains all information related to the required packages for the project. Also, with the help of this configuration files, we can maintain the Project Name and its related version by using the “name” and “version” property

3.angular.json - angular.json file is an Angular Application Environment based JSON file which contains all the information related to the project build and deployment. It tells the system which files need to change when we use ng build or ng serve command.

4.angular.json - This is the main ts file that will run; first, It is mainly used to define the global configurations.

4.styles.css - It is the angular application uses a global CSS.

Components

In Angular, components are the basic building blocks, which control a part of the UI for any application.

A component is defined using the @Component decorator.

Every component consists of three parts, the template which loads the view for the component, a stylesheet which defines the look and feel for the component, and a class that contains the business logic for the component.

Default Code:

{{components_ts}}

@Component Metadata

when we want to create any new component, we need to use the @Component decorator

@Component decorator basically classifies a TypeScript class as a component object

Actually, @Component decorator is a function that takes different types of parameters

The most commonly used properties of the @Component decorator are as follows:
  • selector - It specifies the tag that can be used to call this component in HTML templates just like the standard HTML tags
  • templateUrl - It indicates the path of the HTML template that will be used to display this component.
  • styleUrls - It specifies an array of URLs for CSS style-sheet in the component
  • template - The template is the part of the component which is rendered in the browser. In this property, we can pass the HTML tags or code directly as inline code
  • providers - it is used inject different types of custom services within the component to implement the business logic for the component
Life Cycle of a Component
  • ngOnChanges -This event executes every time a value of an input control within the component has been changed. This event activates first when the value of a bound property has been changed.
  • ngOnInit -This event executed at the time of Component initialization. This event is called only once, just after the ngOnChanges() events. This event is mainly used to initialize data in a component.
  • ngDoCheck -This event is executed every time when the input properties of a component are checked..
  • ngAfterViewInit - This life cycle method executes when the component completes the rendering of its view full.
  • ngOnDestroy - This method will be executed when we want to destroy the Angular components.This hook can be used to clean up the code and detach event handlers.
Data Binding

Types of Data Binding

1.One-way Data Binding

One-way data binding will bind the data from the component to the view (DOM) or from view to the component. One-way data binding is unidirectional.

  • Interpolation
  • Property Binding
  • Event Binding
  • Class Binding
  • Style Binding
Interpolation

The data binding is done from component to the view.Interpolation uses the braces expression

Example

component.ts {{Interpolationts}} component.html {{Interpolationhtml}}

Output

Current Number is {{value1}}
Current Number is {{value1 | currency}}
Current Number is {{dt1}}
Current Number is {{dt1 | date}}
Status is {{status}}
{{status ? "This is correct status" :"This is false status"}}
{{returnString()}}
Property binding

In property binding, we bind a property of a DOM element to a field which is a defined property in our component TypeScript code.

Example

component.ts {{propertyts}} component.html {{propertyhtml}}

Output

Event Binding

Event binding is a data-binding from an element to a component. User actions such as keystrokes, mouse movement, clicks.

It is one-way data binding and sends information from the view to the component class. It is opposite of property binding.

Example

component.html {{eventhtml}} component.ts {{eventts}}

Output

Class Binding

If you wish to add or remove entire classes that are defined in your CSS, you can do this with class binding..

Example

component.html {{classcontent}} component.css {{csscontent}} component.ts {{componentts}}

2.Two-Way Data Binding

In two way data binding, property binding and event binding are combined together.

Two-way data binding in Angular will help users to exchange data from the component to view and from view to the component.

Example

component.html {{twowayhtml}} component.ts {{twowayts}} Output
Enter Your Email
Your email :- {{email}}
Directives

Directives are instructions in the DOM. They specify how to place your components and business logic in the Angular.

Directives are js class and declared as @directive. There are 3 directives in Angular.

  • Component Directives
  • Structural Directives
  • Attribute Directives
Component Directives

It forms the main class and is declared by @Component. It contains the details on component processing, instantiated and usage at run time

    Pipes

    Pipes are a useful feature in Angular. These are the simple way to transform values in an Angular template. It takes the integers, strings, array, and dates as input separated with | to be converted in the format as required and display same as an in the browser.

    Syntax:

    {{pipessyntax}}

    Examples

    app.component.ts {{pipes_ts_string}} app.component.html {{pipes_html_string}}

    Demo

    Date Pipes
    Full Date : {{todayDate}}
    Short Date : {{todayDate | date:'shortDate'}}
    Medium Date : {{todayDate | date:'mediumDate'}}
    Full Date : {{todayDate | date:'fullDate'}}
    Time : {{todayDate | date:'HH:MM'}}
    Time : {{todayDate | date:'hh:mm:ss a'}}
    Time : {{todayDate | date:'hh:mm:ss p'}}

    Number Pipes
    No Formatting : {{amount}}
    2 Decimal Place : {{amount |number:'2.2-2'}}

    Currency Pipes
    No Formatting : {{amount}}
    USD Doller($) : {{amount |currency:'USD':true}}
    USD Doller : {{amount |currency:'USD':false}}
    INR() : {{amount |currency:'INR':true}}
    INR : {{amount |currency:'INR':false}}

    String Related Pipes
    Actual Message : {{message}}
    Lower Case : {{message | lowercase}}
    Upper Case : {{message | uppercase}}

    Percentage Pipes
    2 Place Formatting : {{amount | percent :'.2'}}
